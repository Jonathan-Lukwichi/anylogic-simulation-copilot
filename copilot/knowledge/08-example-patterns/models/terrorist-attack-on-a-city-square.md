# Pattern card — Terrorist Attack On A City Square
- **Source:** AnyLogic example (models) — distilled, original wording
- **Paradigm:** Hybrid (DES + ABM — Pedestrian Library flowchart combined with an autonomous Terrorist agent)
- **Problem it solves:** Model how a vehicle-ramming attack on a public square propagates through a pedestrian crowd, capturing evacuation behaviour and casualty outcomes.

## Block chain
Pedestrians are generated by a **PedSource** at a configurable arrival rate and routed through the square via a series of **PedGoTo** blocks, with **SelectOutput** junctions directing them to different areas or exits. When the terrorist event is triggered, each person's statechart evaluates proximity to the threat and transitions from *alive* to *dead* or routes the agent toward a **PedExit**. The Terrorist is a separate ABM agent whose own statechart begins in a *driving* state; once it enters the pedestrian space, the shared 2-D environment couples the two model layers — the pedestrian flowchart reacts to the terrorist agent's position. Surviving pedestrians leave through **PedExit** or **Exit** blocks; casualties are removed via **PedSink** (the `deadOrAlive` SelectOutput routes them accordingly).

## Resources
No traditional service-resource pools. Capacity is spatial: the city-square environment constrains pedestrian movement. The terrorist vehicle is a single agent (lorry 3-D asset) that acts as a moving hazard rather than a server.

## Key settings worth copying
- **Time unit:** Second — fine-grained enough to capture a fast-moving incident.
- **Pedestrian destination selection:** `exits.get( uniform_discr( exits.size()-1 ) )` — uniform random exit choice from a list, a clean pattern for multi-exit dispersal.
- **Statechart-based fate:** Each `Person` agent carries a two-state statechart (`alive` / `dead`); the transition fires when the person is within lethal range of the terrorist and the terrorist is in the `driving` state (`main.terrorist.inState(Terrorist.driving)`). This guard condition is the coupling point between the two model layers.
- **Terrorist statechart:** Two primary states (`driving` → final), with a transition triggered by entering the pedestrian zone; no distribution-based timing — the event is deterministic once the vehicle is launched.

## KPIs instrumented
- **Casualties:** count of Person agents in the `dead` state, tracked via the `deadOrAlive` SelectOutput routing logic.
- **Evacuees:** persons reaching PedExit/Exit blocks (implicit throughput counter).
- **Crowd density:** visualised through the pedestrian library's built-in density map on the shared space.

## Reusable idea
The key transferable trick is **space as the integration bus for a hybrid model**: instead of explicit message-passing between a pedestrian DES flowchart and an ABM agent, both layers share the same 2-D environment. The ABM agent's position is queried directly inside pedestrian statechart transition guards (`inState` + proximity check), giving a zero-overhead coupling that scales to any scenario where a moving autonomous entity (vehicle, fire, robot) must influence a crowd-flow model without redesigning either sub-model.
