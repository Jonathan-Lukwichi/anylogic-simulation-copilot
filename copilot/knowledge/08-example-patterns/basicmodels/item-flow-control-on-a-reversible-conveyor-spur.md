# Pattern card — Item Flow Control on a Reversible Conveyor Spur
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES
- **Problem it solves:** Controlling item routing at a conveyor intersection where a side spur can buffer, reverse, and drain items independently of the main line.

## Block chain
Items are generated by a Source and travel along a main conveyor (Convey). At the intersection a SelectOutput block decides whether accumulation mode is active. When accumulation is off, items pass straight through additional Convey segments to a Sink. When accumulation is on, items are diverted onto a reversible buffer conveyor (Convey marked as reversible). The first item that enters the spur triggers a direction change so the spur runs away from the main line, filling the buffer. Once the buffer reaches capacity, a PositionOnConveyor markup element at the entry point (spurEnter) calls block() to halt further feed from the main line. A separate "release" action unblocks the spur exit, drains the buffer back toward the main conveyor, and keeps the main-line feed blocked via a releasingFlag until the last item has cleared.

## Resources
No ResourcePool or staff agents. Capacity is governed by the physical length of the reversible conveyor segment (buffer capacity limit set as an integer parameter).

## Key settings worth copying
- **Time unit:** Seconds
- **Reversible conveyor flag:** each Convey block that acts as the spur has its `reversible` property enabled, allowing runtime direction reversal
- **Priority parameter** on ConveyorPathDescriptor controls which path wins when an item arrives at the intersection
- **PositionOnConveyor markup elements** (spurEnter, spurExit) carry custom Java code — direction changes and block/unblock calls live here, not in the flowchart blocks
- **Guard condition:** direction cannot flip while any item is mid-transfer; code checks `spurEnter.getAgent() != null` before switching
- **releasingFlag** boolean prevents the main feed from reopening while the buffer drains

## KPIs instrumented
- Visual color indicator on spurEnter: green when unblocked, red when blocked (used as a live status light in the model canvas)
- Implicit throughput visible from item counts reaching the Sink

## Reusable idea
Place PositionOnConveyor markup sentinels at every decision point on a conveyor network and embed all intersection logic (block, unblock, direction reversal, flag checks) inside those sentinel actions rather than in separate flowchart blocks — this keeps the Material Handling Library layout clean while allowing arbitrarily complex custom control logic at each physical junction.
