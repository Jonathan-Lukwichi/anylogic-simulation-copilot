# Pattern card — Lead Acid Battery Production Phase 5
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES
- **Problem it solves:** Models a multi-stage lead-acid battery manufacturing line that combines conveyor movement, transporter (forklift) logistics, crane operations, batching, unbatching, and final assembly to find throughput bottlenecks.

## Block chain
Electrode plates are generated by two Source blocks (one per plate type) and immediately grouped into batches (Batch). Batches ride conveyors (Convey / ConveyorEnter) to a staging queue, then a forklift fleet (TransporterFleet + MoveByTransporter) carries them to processing stations where they undergo timed operations (Delay). After processing, plates are unbatched (Unbatch), travel via conveyor to an Assembler block that combines the two electrode streams with a separator stream into a complete battery block (BatteryBlock agent). Finished blocks are lifted by a jib crane (MoveByCrane using batteryBlockCrane) and placed on an outfeed conveyor. A second Assembler merges battery blocks with case components sourced from another Source. Hold blocks gate flow until upstream operations complete. Final assembled batteries exit through a Sink.

## Resources
- **TransporterFleet** — forklift pool (Forklift agents); capacity set by fleet size parameter.
- **ResourcePool** — Operator agents; capacity defined via schedule or fixed value.
- **Crane** — one industrial jib crane (batteryBlockCrane, JibCraneDescriptor) shared across MoveByCrane steps.
- **Conveyors** — multiple fixed-speed conveyor segments with individual capacity settings.

## Key settings worth copying
- Time unit is **seconds**, keeping conveyor speeds and delay durations in SI units.
- Batch size is parameterised on the PlatesBatch agent class — change one value to rescale the whole line.
- Forklift and crane are modelled as separate resource types; seize/release is handled implicitly by MoveByTransporter and MoveByCrane, so no explicit Seize/Release blocks are needed.
- ConveyorEnter blocks are used where agents join a moving conveyor mid-line rather than at the conveyor start — useful for merging streams onto a shared belt.
- Hold blocks act as synchronisation gates, releasing agents only when a downstream condition is satisfied.

## KPIs instrumented
- Throughput (batteries completed per shift) tracked at the final Sink.
- Utilisation of the forklift fleet and operator resource pool.
- Queue length at the Assembler input ports (signals whether plate supply or case supply is the constraint).
- Conveyor content (number of items in transit on each belt segment).

## Reusable idea
Use a pair of Assembler blocks in series — the first merges two manufactured sub-components (electrode blocks), and the second merges that sub-assembly with a purchased component (battery case) — to model any two-stage bill-of-materials join without writing custom synchronisation logic.
