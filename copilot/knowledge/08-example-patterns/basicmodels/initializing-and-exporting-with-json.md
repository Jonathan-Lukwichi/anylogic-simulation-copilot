# Pattern card — Initializing and Exporting with JSON
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** Hybrid (ABM + SD elements within agent populations)
- **Problem it solves:** Demonstrates bidirectional JSON data exchange between an AnyLogic model and Python via the Pypeline add-on, covering population initialization from JSON and selective state serialization back to JSON files.

## Block chain
The model contains a `Main` agent that holds a population of `CompanyStock` agents. Each `CompanyStock` agent internally uses Stock/Flow variables to simulate a simple normalized stock price driven by a volatility-based flow. A `PyCommunicator` object in `Main` mediates all Python interaction. On a periodic schedule, the model serializes the `stocks` population to a JSON file (e.g., `stocks-2020Q3.json`) using `pyCom.toJson(...)`. On demand, the model can reinitialize the population from a JSON file produced by Python (`pyCom.fromPopulationJson(..., CompanyStock.class, stocks)`), or it can accept a raw JSON string returned directly from a Python call.

## Resources
- One `PyCommunicator` instance (Pypeline library block) acting as the Python bridge
- Agent population: `stocks` — a collection of `CompanyStock` agents, each with its own stock-price Stock variable and price-change Flow variable

## Key settings worth copying
- **Time unit:** Day
- **Price initialization:** `uniform(0.3, 0.7)` — each stock gets a random normalized starting price
- **Volatility (shares outstanding proxy):** `triangular(2, 10, 5)` — triangular distribution with min 2, max 10, mode 5
- **Price movement direction:** `uniform() >= modLookup(stockPrice) ? 1 : -1` — probabilistic sign flip based on current price level
- **Serialization filter:** `pyCom.jsonFilter` is configured to exclude auto-generated datasets (`.includeAutogenerated()` toggled off) and can include/exclude by class or package — keeping output compact and readable
- **Output filename pattern:** `String.format("stocks-%4dQ%d.json", getYear(), floorDiv(getMonth(), 3)+1)` — quarterly file naming derived from simulation calendar time

## KPIs instrumented
- Normalized stock price time-series chart for all `CompanyStock` agents displayed on the Main dashboard
- Console trace of the full JSON state string (via `traceln(json)`) for debugging serialization output

## Reusable idea
Use `PyCommunicator.fromPopulationJson(...)` to hot-swap an entire agent population's parameter values at runtime from externally computed JSON — this lets a Python script (e.g., an optimizer or data pipeline) drive AnyLogic scenario inputs without rebuilding the model, and `toJson(...)` with a filter closes the loop by exporting only the state fields that matter to downstream analysis.
