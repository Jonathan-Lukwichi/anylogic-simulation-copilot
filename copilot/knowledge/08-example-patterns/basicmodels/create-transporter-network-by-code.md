# Pattern card — Create Transporter Network by Code
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES
- **Problem it solves:** Shows how to build a transporter network entirely through Java code at startup rather than through the GUI, enabling dynamic or data-driven layout construction.

## Block chain
Agents are generated by a **Source** block and immediately handed to a **MoveByTransporter** block. A **TransporterFleet** owns one or more **Transporter** agents that navigate the network to pick up each entity. The transporter travels from a **PointNode** (pickup location) along two joined **Path** segments — bridged by a pair of **NetworkPort** objects — into a **RectangularNode** (destination area), then continues inside the rectangular node to a specific **Attractor**. Once delivered, the entity flows to a **Sink**. The entire level, nodes, paths, ports, and attractors are instantiated inside `createLevelAndNetwork()`, a startup method called before the first agent arrives.

## Resources
- 1 TransporterFleet with at least 1 Transporter agent (speed: 10 m/s, body dimensions: 1 m × 1 m × 1 m)
- 1 PointNode (pickup)
- 1 RectangularNode (500 × 500 px, 100 × 100 cell grid) as destination
- 2 Paths joined via NetworkPorts to form a single logical route

## Key settings worth copying
- Transporter speed: **10 MPS** (meters per second)
- RectangularNode grid: **100 columns × 100 rows**, 500 × 500 px canvas size
- Path stitching: two separate Path objects share a NetworkPort pair so pathfinding treats them as one continuous corridor
- Model time unit: **Second**
- Network and Level objects are variables (`n`, `myLevel`) populated entirely in startup code via `createLevelAndNetwork()`

## KPIs instrumented
- n/a (demonstration model — no explicit KPI dashboards; focus is structural correctness of the programmatically built network)

## Reusable idea
Build your material-handling network in code, not on the canvas — store node and path references in variables, wire them with `path.setSource(pn)` / `path.setTarget(rn)` calls, and add NetworkPorts where paths must join. This pattern lets you generate layouts from external data (CAD coordinates, database records, parameter files) without touching the GUI, making it ideal for scenario automation or layout optimisation loops.
