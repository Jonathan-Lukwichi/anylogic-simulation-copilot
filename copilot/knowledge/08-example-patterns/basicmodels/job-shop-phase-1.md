# Pattern card — Job Shop - Phase 1
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES
- **Problem it solves:** Models pallets arriving at a job shop, being stored in a raw-material rack, processed on a machine, then retrieved and dispatched — demonstrating rack storage with forklift transport in a basic material-handling loop.

## Block chain
Pallets are generated by a Source (interarrival-time mode, 5-minute fixed interval) and immediately directed to a Store block that places each pallet into a rack location for raw material. A forklift from the shared resource pool is seized to perform the put-away move. When the downstream machine is ready, a Retrieve block pulls the pallet from the rack (again using the forklift) and feeds it into a Delay block that simulates the machining operation with a triangular(15, 20, 30) minute duration. After processing a second Retrieve/Sink pair removes the finished pallet from the system.

## Resources
- **Forklifts:** a single transporter resource pool (`forklifts`) with a home node at `forkliftParking`; used for both inbound store moves and outbound retrieve moves.
- **Rack storage:** a physical rack (`storeRawMaterial`) with defined cell capacity; acts as a buffer between arrival and processing.

## Key settings worth copying
- Arrival type: `INTERARRIVAL_TIME`, fixed at **5 minutes** (model time unit = Minute)
- Entity type: `Pallet` agent (imported from the Phase 2 package, showing cross-package reuse)
- Machining delay: `triangular(15, 20, 30)` minutes — asymmetric service time with a realistic spread
- Transporter home node: `forkliftParking` — forklifts return to a designated parking area after each task
- Rack cell capacity is explicitly parameterised, making it easy to vary storage size experimentally

## KPIs instrumented
Phase 1 is a structural foundation model; no explicit KPI dashboards are wired in this phase. Utilisation of the forklift pool and rack occupancy are the natural candidates to instrument in subsequent phases.

## Reusable idea
Use a **Store → Delay → Retrieve** sandwich with a shared transporter pool to model any rack-based buffer: the forklift is seized automatically by Store/Retrieve blocks, keeping routing logic clean and separating transport from processing without manual Seize/Release blocks.
