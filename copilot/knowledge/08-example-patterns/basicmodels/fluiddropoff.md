# Pattern card — FluidDropoff
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** Hybrid (DES + Fluid)
- **Problem it solves:** Models truck agents delivering bulk liquid/granular material to a storage tank via a conveyor, combining discrete agent movement with continuous fluid flow.

## Block chain
Trucks are generated by a **Source** and dispatched using **MoveTo** blocks to travel to the dropoff point. Upon arrival, a **FluidDropoff** block transfers fluid from the truck's onboard capacity onto a **BulkConveyor**, which transports the material continuously to a **Tank**. A **Valve** controls outflow from the tank (e.g., to simulate consumption or drainage), and excess or spent fluid exits through a **FluidDispose** block. Trucks then move on (another **MoveTo**) and exit at a **Sink**.

## Resources
- Tank (fluid storage with configurable capacity)
- BulkConveyor (continuous belt carrying fluid between dropoff and tank)
- Valve (controls tank outlet flow rate)
- Truck agent population (discrete carriers of fluid load)

## Key settings worth copying
- Time unit: **Second** — appropriate for short-cycle logistics/fluid transfer simulations
- `rate` parameter on **FluidDropoff** — sets the volumetric transfer rate from truck to conveyor
- `rate` parameter on **Source** — controls truck arrival frequency
- BulkConveyor speed and capacity define how quickly material reaches the tank
- Valve open/close logic governs tank drainage rate independent of inflow

## KPIs instrumented
- Tank fluid level over time (content)
- Conveyor utilisation / load
- Truck cycle time (arrival → dropoff → departure)
- Throughput (volume delivered per unit time)

## Reusable idea
The key transferable trick is decoupling discrete agent logic (truck routing via MoveTo) from continuous fluid dynamics (FluidDropoff → BulkConveyor → Tank) within the same model: agents trigger fluid transfers at precise moments, letting you model bulk-material logistics without discretising the fluid itself.
