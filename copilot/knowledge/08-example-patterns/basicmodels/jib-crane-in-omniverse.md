# Pattern card — Jib Crane in Omniverse
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES
- **Problem it solves:** Demonstrates how to co-simulate a material-handling crane in AnyLogic while streaming live 3-D state to an NVIDIA Omniverse scene via the Omniverse Connector.

## Block chain
Boxes are generated by a Source block at a constant arrival rate. Each box agent is then routed through a MoveByCrane block that commands the jib crane to pick up and transport the load to a destination point. A second MoveByCrane block handles any return or secondary movement. Completed loads are consumed by a Sink block. The crane object is defined by a JibCraneDescriptor and referenced by both MoveByCrane blocks so they share the same physical resource.

## Resources
One jib crane instance (JibCraneDescriptor attached to a USD scene node at `/World/jib_crane`). No explicit ResourcePool — the crane descriptor itself serialises access between the two MoveByCrane steps.

## Key settings worth copying
- **Time unit:** Seconds (fine-grained enough for crane motion).
- **Arrival rate:** Constant rate parameter on the Source block (no stochastic distribution used — deterministic throughput test).
- **USD scene path:** `omniverse://localhost/Users/admin/JibCraneAssets/jib_crane_scene.usd` — must be copied to an Omniverse Nucleus server before linking.
- **UsdUseGeneratedObjects:** true — AnyLogic auto-publishes agent transforms to the USD stage each simulation step.
- **OmniverseConnector:** single connector element wiring the AnyLogic model to an Omniverse Live Session; path prefix must start with `omniverse://`.

## KPIs instrumented
n/a — this model is a visualisation/integration demo rather than a performance-measurement model; no wait-time, utilisation, or throughput statistics are explicitly collected.

## Reusable idea
The transferable trick is the **USD live-sync pattern**: assign a `JibCraneDescriptor` (or any AnyLogic material-handling descriptor) a matching USD prim path, enable `UsdUseGeneratedObjects`, point the OmniverseConnector at a Nucleus-hosted `.usd` root scene, and AnyLogic will push position/rotation updates in real time — giving photorealistic 3-D feedback without rebuilding the simulation logic in a separate tool.
