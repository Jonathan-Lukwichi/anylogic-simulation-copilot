# Pattern card — Buffer with Lift
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES
- **Problem it solves:** Routing mixed-type items from an inbound conveyor into the correct buffer lane via a priority-driven vertical lift, then releasing full batches to an outbound conveyor.

## Block chain
Items are generated by a Source and placed onto a feed-in conveyor. A Convey block hands each item to a SelectOutput router that inspects the item's type attribute and directs it toward the appropriate BufferLine lane. A Lift markup element (bufferInputLift) picks up the first item from whichever batch has the highest priority, then collects the remaining items of that same batch before serving the next priority group — preventing interleaving of different batches on the lift car. Each BufferLine accumulates exactly five boxes; once that threshold is met, the lane releases the full batch onto an outgoing conveyor (Convey) that carries items to a Sink. A second lift (bufferOutputLift) handles the descent back to the ground-level output conveyor.

## Resources
- Two Lift markup elements: bufferInputLift (inbound, priority-ordered) and bufferOutputLift (outbound)
- Multiple BufferLine instances (one per item type / priority tier), each with its own conveyor segment
- ConveyorEnter (placeOnConveyor) entry point shared by all lanes merging onto the output conveyor

## Key settings worth copying
- **Batch accumulation threshold:** 5 boxes per BufferLine before release
- **Lift dispatch rule:** priority field on the agent — the lift selects the highest-priority batch first, then exhausts that batch completely before moving on (no premature switching)
- **Routing logic:** SelectOutput evaluates `agent.priority` and indexes into `bufferLines.get(agent.priority).conveyor` to find the target conveyor for that lane
- **Time unit:** Seconds
- **Arrival process:** rate-based Source (interarrivalTime parameter); no explicit distribution shown — default exponential implied by the rate parameter

## KPIs instrumented
- Throughput at Sink (items cleared per time unit)
- Buffer lane fill level (how quickly each lane reaches its batch of 5)
- Lift utilisation (fraction of time the lift car is moving vs. idle waiting for the next priority batch)

## Reusable idea
Model a vertical lift (or any single-lane sorter) as a priority-batch server: always pick the highest-priority class first, but commit to completing the entire batch of that class before switching — this minimises costly direction reversals and head-of-line blocking, and is directly replicable whenever a shared transport resource must serve multiple queues without interleaving.
