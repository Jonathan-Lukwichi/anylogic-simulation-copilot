# Pattern card — Lead Acid Battery Production Phase 3
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES
- **Problem it solves:** Models a multi-stage lead-acid battery assembly line that combines conveyor material flow, pallet-based batching, forklift transport, and operator resources to find throughput bottlenecks.

## Block chain
Electrode plates are generated by a Source and travel on conveyors (Convey) to a pasting/drying stage. Plates are grouped into pallets using Batch blocks, then forklifts (MoveByTransporter) carry pallets between work areas. At each processing station a Delay block represents the machine cycle time. After processing, Unbatch blocks dissolve pallets back into individual plates. Two streams of plates (positive and negative electrodes) converge at an Assembler block that combines them into a battery block, which then rides a final conveyor to the Sink. A second Assembler joins battery blocks into complete battery cases before exit.

## Resources
- **ResourcePool** — operators (sitting workers) assigned to assembly stations
- **TransporterFleet** — forklift agents that carry batched pallets between nodes on a routed network
- Conveyor segments each carry a defined capacity and speed; area nodes act as transfer points

## Key settings worth copying
- Time unit: **seconds** (fine-grained enough to capture conveyor travel and short cycle times)
- **Batch** block configured with a fixed `batchSize` parameter (number of plates per pallet)
- **MoveByTransporter** used instead of a simple Delay to model realistic forklift travel paths between AreaNodeDescriptors
- Conveyor stations defined as `ConveyorSimpleStationDescriptor` with per-segment speed and capacity restrictions
- **Assembler** requires two separate input ports (positive + negative plate streams) before releasing one combined output — models true part synchronisation
- 3-D assets (.dae files) attached to agents for visual validation of flow on a factory layout background image

## KPIs instrumented
- Throughput measured at the Sink (completed batteries per shift)
- Utilisation of the TransporterFleet (forklift idle vs. busy time)
- Queue content at each Batch block (pallet accumulation before transport)
- ResourcePool utilisation for operators at assembly stations

## Reusable idea
**Batch-transport-Unbatch sandwich around MoveByTransporter:** when individual parts must travel together as a unit (on a pallet, in a tray, in a bin), wrap MoveByTransporter between a Batch and an Unbatch rather than moving agents one by one. This keeps forklift trips realistic, collapses per-part overhead, and lets you vary pallet size as a single parameter to explore throughput vs. handling trade-offs.
