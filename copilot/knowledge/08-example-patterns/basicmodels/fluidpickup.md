# Pattern card — FluidPickup
- **Source:** AnyLogic example (basicmodels) — distilled, original wording
- **Paradigm:** DES (Fluid / Material Handling hybrid)
- **Problem it solves:** Models discrete agents (vehicles/cars) visiting a fluid source to pick up a measured volume of bulk material, then departing — bridging the gap between agent-based movement and continuous fluid flow.

## Block chain
Agents (Car) are generated by a **Source** block and routed via **MoveTo** blocks to a designated pickup location. At that location, a **FluidPickup** block attaches the agent to a fluid network consisting of a **Tank** (bulk storage), a **Pipeline** (transfer conduit), and a **Valve** (flow control). The FluidPickup draws a specified volume from the Tank through the Pipeline; the Valve governs when flow is permitted. Once the required amount is loaded, the agent travels away via another **MoveTo** and exits through a **Sink**.

## Resources
- **Tank** — bulk fluid storage with configurable capacity and initial fill level (units: cubic metres)
- **Pipeline** — connects Tank to FluidPickup; governs transfer flow rate (cubic metres per second)
- **Valve** — opens/closes to control fluid transfer; can be triggered by agent arrival events
- Multiple parallel FluidPickup stations are present, each served by its own Pipeline/Valve/Tank chain

## Key settings worth copying
- Fluid amounts measured in **CUBIC_METER**; flow rates in **CUBIC_METER_PER_SECOND**
- FluidPickup parameter `pickupByAmount` set to a fixed volume (`fluidToPickup`) rather than time-based fill
- `pickupPoint` network node used as the spatial anchor for MoveTo routing
- Model time unit: **Second**
- A commented-out line (`uniform_discr(16)`) hints at randomising batch size — easy to activate for stochastic loading times
- Car agent population typed as a custom `Car` class, allowing per-agent state tracking during loading

## KPIs instrumented
- Fluid level remaining in each Tank (continuous)
- Agent throughput at Sink (vehicles served per time unit)
- Loading time per vehicle (time between FluidPickup entry and exit)
- Tank `amountAbove` / `amountBelow` threshold events for replenishment triggers

## Reusable idea
The core transferable trick is using **FluidPickup as a bridge between discrete agents and a continuous fluid network**: an agent arrives at a node, the Valve opens, fluid drains from Tank through Pipeline into the agent's "container", the Valve closes when the target volume is reached, and the agent departs — all without downloading any fixed-duration delay, making loading time dynamically dependent on actual flow rate and available stock.
